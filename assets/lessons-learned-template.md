# PM 踩坑记录（Lessons Learned）

> **使用说明**
> 将此文件复制到 `pm-workspace/` 目录，重命名为 `lessons-learned.md`。
> 每次需求评审、产品内审或研发联调后，把被指出的问题记录在这里。
> PM Copilot 在进行 Step 7 讨论时会自动读取此文件，帮你避免重蹈覆辙。

---

## 记录格式

每条记录包含以下字段：

```
### [简短标题]
- **发现时间：** YYYY-MM-DD
- **所属需求：** [需求名称]
- **评审场合：** [产品内审 / 需求评审 / 研发联调 / 上线复盘]
- **问题描述：** 当时写了什么或遗漏了什么
- **收到的反馈：** 评审时被指出的具体问题
- **正确做法：** 应该怎么写/怎么想
- **分类标签：** [模块划分 / 业务规则遗漏 / 技术细节混入 / 异常流缺失 / 验收标准模糊 / 其他]
```
